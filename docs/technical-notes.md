# 技术说明

## 1. 可见光通信

通信仿真使用 OOK 强度调制。数据帧由前导码、11 位 Barker 帧同步码、帧头、
载荷和 XOR 校验组成。接收端在单个码元周期内对采样值进行积分，并根据接收
信号统计特性生成自适应判决阈值，以降低随机噪声和光强变化造成的误判。

信道模型采用 Lambertian LED 辐射模型，并考虑光电探测器面积、响应度、
视场角、发射距离和环境光。噪声由散粒噪声与热噪声组成。

`vlc_ook_gui.mlx` 支持文本和图像两种载荷。较大的数据会被拆分为多个数据包，
每包包含序号、总包数、载荷长度和校验和。

## 2. 可见光定位

系统在室内布置 4 个 LED，每个 LED 对应独立的信号特征。离线阶段在参考点
计算来自不同 LED 的 RSS，形成坐标与 RSS 向量的映射。

在线阶段计算待定位 RSS 向量与指纹库之间的欧氏距离，选取距离最小的 K 个
参考点，并使用距离倒数作为权重估计二维坐标。

```text
输入 RSS 向量
    -> 与指纹库计算欧氏距离
    -> 选择 K 个近邻
    -> 距离倒数加权
    -> 输出二维估计位置
```

## 3. 信道与障碍物

定位脚本计算 LED 到接收点的 LOS 增益，并通过离散墙面单元估计一次反射
NLOS 增益。障碍物会被标记为不可通行区域，同时用于判断部分直射路径是否
被遮挡。

## 4. 航迹规划

定位结果被转换为 `30 m × 20 m` 仓库中的 `0.5 m` 分辨率栅格地图起始节点。
A* 使用 8 邻域扩展，
累计路径长度作为实际代价 `g(n)`，到目标点的欧氏距离作为启发代价 `h(n)`，
搜索避开障碍物的可行路径。

## 5. 项目边界

仿真完成了通信、定位和规划模块的联合展示。硬件原型验证了光脉冲通信和
多频 LED 信号分离，但未实现无人机飞控接入、实机定位闭环或动态障碍物规避。
