# 硬件原型说明

## 通信原型

发送端使用 STM32、4×4 矩阵键盘和 LED。按键值被编码为光脉冲数量，
通过 DWT `CYCCNT` 实现微秒级脉冲延时，并使用 SysTick 完成周期任务调度。

接收端持续采集光敏器件输出，根据稳定光照阶段更新参考值。当采样值相对参考值
出现明显下降时，将其识别为光脉冲，并在时间窗口内完成脉冲计数。

## 多光源识别原型

发射端通过 STM32 定时器输出四路不同频率 PWM 驱动 LED。接收端使用
OPT101 将光信号转换为电压，并通过 ADC 和 DMA 以 `120 kHz` 采集
`1000` 个采样点。

采样数据通过 USART 发送至 MATLAB，上位机依次执行：

1. 根据帧头与帧尾解析完整数据帧。
2. 减去采样均值，去除环境光造成的直流分量。
3. 使用 Hann 窗降低频谱泄漏。
4. 执行 FFT，并在目标频率附近搜索局部峰值。
5. 将各频率峰值幅度作为不同 LED 的信号特征。

## 已验证与未验证内容

已完成基础光脉冲通信、ADC/DMA 数据采集以及多频 LED 频域分离验证。
当前仓库不包含原 STM32 工程源码，也未完成无人机飞控或硬件端 A* 路径规划。
